# 🐟 Salmon — мультиклиентское PWA (один сайт, много клиентов)

Один репозиторий и **один сайт на GitHub Pages** обслуживают всех клиентов.
Клиент выбирается параметром ссылки `?client=<id>`. У каждого клиента — **свой
бэкенд** (своя Google-таблица + свой Apps Script + свой URL). Вся бизнес-логика
в Apps Script; фронтенд — «тупая» оболочка, которая строит интерфейс по ответу
сервера.

```
…/tradingmind/?client=salmon     → клиент Salmon  (свой Apps Script)
…/tradingmind/?client=romashka   → клиент Ромашка (свой Apps Script)
```

| Роль | Что может |
|------|-----------|
| **admin**  | форма ввода + «Отправить» + «Загрузить» |
| **viewer** | только «Загрузить» (чтение) |

Роль, разрешённые действия, схему формы и **уже отформатированные** данные решает
и присылает сервер. Токены вводит пользователь (в `localStorage`, отдельно на
каждого клиента) — **в коде токенов нет**.

## 📁 Состав

```
webapp/
├── index.html          # тонкая оболочка (без захардкоженных полей)
├── style.css           # мобильная тема (цвета берутся из конфига клиента)
├── app.js              # ЕДИНЫЙ для всех клиентов — НЕ правится под клиента
├── configs/            # по файлу на клиента (только публичные параметры!)
│   ├── _template.js    # шаблон для нового клиента
│   ├── salmon.js
│   └── romashka.js
├── manifest.json       # статичный fallback (app.js подменяет на пер-клиентский)
├── service-worker.js   # офлайн-кэш (относительные пути, scope ./)
├── .nojekyll           # отключает обработку Jekyll на Pages
├── icons/              # иконки PWA
└── code.gs             # бэкенд — копия у каждого клиента (вся логика тут)
```

### Архитектурные принципы
- **Тонкий фронт.** Никаких расчётов/валидации/форматирования/прав в браузере.
  Фронт читает `?client=`, шлёт токен, рисует ответ сервера.
- **Вся логика в Apps Script.** Проверка токена, роль, валидация, вычисления,
  форматирование и решение «что показывать» — на бэкенде.
- **Различие клиентов — только данные.** Всё клиентское вынесено в `configs/<id>.js`
  (имя, `apiUrl`, тема). `app.js` одинаков у всех.
- **GitHub Pages.** Все пути относительные (работает в подкаталоге `/repo/`).
  PWA: `app.js` собирает manifest на лету с `start_url` `./?client=<id>`, чтобы
  установленное приложение помнило клиента.

## 🚀 Первичная настройка (один раз)

1. Залейте папку `webapp/` в публичный репозиторий и включите **Pages**
   (*Settings → Pages → Deploy from a branch → main → /(root)*).
   Репозиторий **публичный** — поэтому в `configs/` только публичные значения,
   без токенов.
2. Базовый адрес: `https://USERNAME.github.io/REPO/`.

> Бэкенд (`code.gs`) на Pages не публикуется — он живёт в Apps Script каждого
> клиента. В репозитории он лежит как эталон для копирования.

## ⏱️ Подключить нового клиента за 10 минут (новый репозиторий НЕ создаём)

Пример: клиент `romashka`.

**Бэкенд (его собственный):**
1. Скопируйте таблицу-шаблон `Salmon.xlsx` в Google Sheets (отдельную для клиента).
2. *Расширения → Apps Script* → вставьте **`code.gs`** → сохраните.
3. Задайте токены: *⚙ Настройки проекта → Свойства скрипта*:
   `имя = токен`, `значение = admin` или `viewer`.
   (Можно один раз запустить `seedTokens()`, потом очистить; `makeToken()` —
   генератор случайного токена в журнал.)
4. *Развернуть → Новое развёртывание → Веб-приложение*: запуск **от меня**,
   доступ **Все**. Скопируйте URL `…/exec`.

**Фронтенд (одна правка):**
5. Скопируйте `configs/_template.js` → `configs/romashka.js`, впишите:
   ```js
   registerClient({
     id: 'romashka',
     name: 'Ромашка',
     apiUrl: 'https://script.google.com/macros/s/XXXX/exec',
     theme: { accent: '#7c3aed', accent2: '#a78bfa' }
   });
   ```
6. Закоммитьте/запушьте файл. **Никакие другие файлы не трогаем.**
7. Дайте клиенту ссылку: `https://USERNAME.github.io/REPO/?client=romashka`.

> Подключение клиента = **один файл в `configs/`** + **Script Properties** его
> бэкенда. Ни нового репозитория, ни нового деплоя, ни правок `app.js`.

### ✅ Чек-лист проверки
- [ ] Открыли `…/?client=romashka` — видно имя «Ромашка», экран входа.
- [ ] Открыли `…/?client=НЕСУЩЕСТВУЕТ` — понятная заглушка «Неизвестный клиент».
- [ ] Открыли без `?client=` — заглушка с подсказкой про параметр.
- [ ] Вход admin-токеном → видны форма + «Загрузить».
- [ ] Вход viewer-токеном → только «Загрузить».
- [ ] Неверный токен → «Неверный токен».
- [ ] «Отправить» создаёт строку в листе «Продажи»; «Загрузить» её показывает.
- [ ] На телефоне ставится на главный экран (PWA), ссылка сохраняет `?client=`.

## 🔌 Контракт API (что возвращает Apps Script)

- `GET ?action=whoami&token=…` →
  `{ ok, role, actions:["submit","load"], form:{title,fields:[…]} }`
- `GET ?action=read&token=…` →
  `{ ok, role, list:{ items:[{title,meta:[…],amount}], summary:[{label,value}] } }`
- `POST {token, action:"create", row:{…}}` (text/plain) →
  `{ ok, message }` либо `{ error:"validation", message }`
- Ошибки: `{error:"unauthorized"}`, `{error:"forbidden"}`.

> POST шлётся как `text/plain` без кастомных заголовков — чтобы не было
> CORS-preflight, который Apps Script не умеет обрабатывать.

## 🧩 Поля продажи

`Дата · Товар · Кол-во · Скидка · Тип скидки (сумма/%) · Клиент · Оплата (счёт/«В долг»)`.
Схему присылает сервер (`whoami.form`), суммы/себестоимость/прибыль считает лист,
а форматирование списка делает Apps Script. Модель совпадает с `Salmon.xlsx`.

> ⚠️ Лист **«Продажи»** должен содержать строку-образец №4 с формулами — бэкенд
> копирует её на каждую новую запись.
