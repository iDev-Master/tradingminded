# 🐟 Salmon — мобильный складской учёт (PWA + Google Apps Script)

Маленькое мобильное веб-приложение для учёта продаж. Фронтенд — статика на
**GitHub Pages**, бэкенд — **Google Apps Script**, данные — в **Google-таблице**.
Вход по токену, две роли:

| Роль | Что может |
|------|-----------|
| **admin**  | вводить новые продажи + загружать список |
| **viewer** | только загружать список (чтение) |

Токены **не хранятся в коде** — admin задаёт их в Script Properties, а пользователь вводит в приложении (токен сохраняется в `localStorage` устройства).

## 📁 Состав

```
webapp/
├── index.html          # разметка (экран входа + экран приложения)
├── style.css           # мобильная тёмная тема
├── app.js              # логика, вход, роли, запросы к бэкенду
├── manifest.json       # PWA-манифест (установка на главный экран)
├── service-worker.js   # офлайн-кэш статики
├── icons/              # иконки PWA (192 / 512)
└── code.gs             # бэкенд — вставляется в Google Apps Script
```

## 🚀 Развёртывание

### Часть 1. Бэкенд (Google Apps Script)

1. Создайте Google-таблицу: <https://sheets.new>. Имя любое.
2. В ней: **Расширения → Apps Script**. Откроется редактор.
3. Удалите содержимое `Code.gs` и вставьте код из файла **`code.gs`** этого проекта. Сохраните (💾).
4. **Задайте токены** (⚙ *Настройки проекта* → *Свойства скрипта* → *Добавить свойство*):
   - Имя свойства = **сам токен** (напр. `a1b2c3d4e5f6`)
   - Значение = `admin` или `viewer`
   - Добавьте столько пользователей, сколько нужно.

   > 💡 Можно вместо этого один раз запустить функцию `seedTokens()` (впишите туда свои токены), а затем очистить её. Случайный токен генерирует функция `makeToken()` — запустите и скопируйте из *Журнала выполнения*.

5. **Опубликуйте как Web App:** *Развернуть* → *Новое развёртывание* → шестерёнка → **Веб-приложение**.
   - *Описание*: любое
   - *Запуск от имени*: **Я** (от вашего аккаунта)
   - *У кого есть доступ*: **Все** (Anyone) — обязательно, иначе фронтенд не достучится.
   - Нажмите **Развернуть**, дайте разрешения.
6. Скопируйте **URL веб-приложения** (вида `https://script.google.com/macros/s/XXXX/exec`).

> ⚠️ После любой правки `code.gs` нужно *Развернуть → Управление развёртываниями → ✎ → Новая версия → Развернуть*, иначе изменения не применятся.

### Часть 2. Фронтенд

7. Откройте **`app.js`** и впишите скопированный URL в первую строку конфига:
   ```js
   const API_URL = 'https://script.google.com/macros/s/XXXX/exec';
   ```

### Часть 3. GitHub Pages

8. Создайте репозиторий на GitHub и залейте туда **содержимое папки `webapp/`** (важно, чтобы `index.html` лежал в корне публикуемой папки).
   ```bash
   cd webapp
   git init
   git add .
   git commit -m "Salmon PWA"
   git branch -M main
   git remote add origin https://github.com/USERNAME/REPO.git
   git push -u origin main
   ```
9. На GitHub: **Settings → Pages** → *Build and deployment* → *Source*: **Deploy from a branch** → ветка `main`, папка `/ (root)` → **Save**.
10. Через минуту откроется адрес `https://USERNAME.github.io/REPO/`. Откройте его на телефоне.

### Часть 4. Установка на телефон (PWA)

- **iPhone (Safari):** *Поделиться* → *На экран «Домой»*.
- **Android (Chrome):** меню ⋮ → *Установить приложение*.

## 🔐 Как это работает

```
[Телефон / PWA]  --GET ?token&action=read-->   [Apps Script]  <-->  [Google Таблица]
                 --POST {token,action,row}-->
```

- **Вход:** приложение шлёт `action=whoami`, бэкенд по токену возвращает роль → интерфейс подстраивается.
- **Отправить (admin):** `POST action=create` → новая строка на лист «Продажи». Сумма считается на сервере.
- **Загрузить (обе роли):** `GET action=read` → список строк в JSON.
- Неверный токен → `{error:"unauthorized"}` → приложение просит войти заново.
- viewer пытается писать → `{error:"forbidden"}`.

### Про CORS у Apps Script
POST отправляется с `Content-Type: text/plain` и без кастомных заголовков —
это «простой запрос», поэтому браузер не делает preflight `OPTIONS`,
который Apps Script не умеет обрабатывать.

## 🧩 Поля продажи

Форма admin отправляет: `Дата · Товар(артикул) · Кол-во · Скидка · Тип скидки (сумма/%) · Клиент · Оплата (счёт/«В долг»)`.
Бэкенд пишет их во вводные колонки листа **«Продажи»**, а формулы листа сами считают
`Наименование · Цена · Сумма · Себестоимость · Прибыль`.

Списки товаров и счетов фронтенд берёт из листа **«Справочники»** (action `whoami`),
поэтому дропдауны всегда актуальны. Модель полностью совпадает с `Salmon.xlsx`
(средневзвешенная себестоимость, мультивалюта, долги — на стороне таблицы).

> ⚠️ Лист **«Продажи»** должен содержать строку-образец №4 с формулами (как в
> `Salmon.xlsx`) — бэкенд копирует её на каждую новую запись.

## 🛠 Частые проблемы

| Симптом | Причина / решение |
|---------|-------------------|
| «Не задан URL бэкенда» | Не вписали `API_URL` в `app.js`. |
| «Неверный токен» | Токен не добавлен в Script Properties или есть лишний пробел. |
| «Нет сети или сервер недоступен» | Доступ к Web App не «Все», либо не пере-развёрнут после правки. |
| Данные не обновляются | Создайте новую версию развёртывания (см. ⚠️ выше). |
| Иконка/манифест не грузятся | Проверьте, что Pages публикует папку с `index.html` в корне. |
